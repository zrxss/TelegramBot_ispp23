# Telegram Bot 

## Что делает бот
Бот перехватывает задания от преподавателя в беседе, выполняет их и складывает в базу данных (откуда потом их можно достать). 
Так же есть функция перегенерации задания, если эталонное решение в базе вам не понравилось, или вы хотите чтобы код выглядел более натурально. 
Для перегенерации сделано 4 разных правдоподобных базовых стиля, которые определяются случайно для вашей перегенерации.
## Как работает обработка файла
Сперва по названию файла определяется его формат (на моем опыте преподаватель скидывает задания только в двух форматах: docx
и txt), если формат не тот, то обработка прекращается, функция заканчивает свою работу. Если форматы те которые нам нужны, мы их скачиваем (docx,
txt), в зависимости от формата разными способами парсим с них текст задания (для docx через библиотеку mammoth, которая позволяет удобно
парсить docx в markdown, сохраняя порядок параграфов и таблиц + LLM любят markdown, для txt просто открываем и считываем файл). Далее так как текст может
быть не заданием, а лекцией, семинаром, каким-либо мусором, мы с помощью запроса в LLM, определяем является ли текст заданием. Если нет,
то функция прекращается и все подчищается. Если да, то генерируется эталонное решение задание и заносится в БД, после чего все так же подчищается.

## Как устроена база данных
База данных обладает следующими полями:
- id INTEGER PRIMARY KEY AUTOINCREMENT - просто ключевое поле, ничего особенного
- date TEXT - дата задания, нужна как для навигации пользователя, так и для работы внутри бота
- task_text TEXT - текст задания, нужно для перегенерации (может быть для будущего функционала тоже)
- solution TEXT - текст эталонного решения задания, которое всегда можно достать по кнопке
- language TEXT - язык для выполнения задания, нужен для улучшения качества промптов, а следовательно решений и перегенераций
- title TEXT - краткое название задания для навигации пользователя в боте

## Как работает личное меню
Бот запускается на команду /start

Пользователю выводится сообщение которое его приветствует и поясняет работу бота.
Снизу появляется клавиатура с функционалом бота:
кнопка "Доступные задания" и кнопка "Перегенерировать ответ".

Кнопка "Доступные задания"

Выводит выбор даты инлайн-клавиатурой за которое пользователь хочет получить задание.
Пользователь выбирает дату и бот высылает все задания за неё.

Кнопка "Перегенерировать ответ"

Выводит выбор даты инлайн-клавиатурой за которое пользователь хочет перегенерировать задание.
Пользователь выбирает дату, бот выводит инлайн-клавиатуру из заданий, по которой можно выбрать
конкретное задание для перегенерации. Пользователь выбирает задание, бот высылает перегенерированное 
решение.


## Как работает генерация и перегенерация
На вход функции отвечающей за генерацию и перегенерацию заданий поступает текст задания, язык на котором
это задание нужно решать, и параметр regen который если True то переводит функцию в режим перегенерации,
иначе функция работает в штатном режиме генерации эталонного решения. В функции определяется промпт посредством
словарей промптов из файла prompts. Потом уже дергается API и функция возвращает решение.

## Переменные конфига
- TOKEN - токен Telegram бота
- GEMINI_KEY - токен API Gemini (не надёжный)
- MISTRAL_KEY - токен API Mistral (годный)
- TEACHER_ID - user id преподавателя в телеграмм, для его блокировки
